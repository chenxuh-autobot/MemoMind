# 设计与启动清单

## 1. 本周先完成的事

### 产品侧

- 明确首版只服务一个主场景
- 确定首版纪要模板
- 确定结构化输出字段
- 明确哪些字段允许 AI 推断，哪些必须保守输出

### 设计侧

- 画 5 个核心页面低保真
- 画“新建纪要 -> 处理中 -> 结果页”的完整流转
- 设计录音中断、上传失败、生成失败的异常状态
- 设计结构化结果的编辑交互

### 技术侧

- 初始化 Android 工程
- 搭好基础架构分层
- 打通 `MNN` 与 Android 的最小 JNI 链路
- 明确首版本地 `Qwen` 模型方案
- 落本地数据库模型
- 定义模型下载与校验机制
- 跑通一条“本地模型生成纪要”的真链路

## 2. 推荐的 10 天节奏

### Day 1-2

- 完成 PRD 精简版
- 定义 Schema
- 画低保真

### Day 3-4

- 初始化 Android 项目结构
- 搭首页、新建页、结果页静态 UI
- 定义数据实体
- 接入本地模型管理骨架

### Day 5-6

- 接入图片选择、录音、文本输入
- 打通本地任务创建与缓存
- 打通本地 `Qwen` 推理调用

### Day 7-8

- 跑通图片/音频预处理到本地纪要生成
- 做本地失败后的云端辅助兜底

### Day 9

- 优化异常流
- 优化结果可读性

### Day 10

- 完成一版演示脚本
- 准备参赛展示材料

## 3. 需要尽快拍板的设计决策

1. 首版主场景到底选哪一个：
   头脑风暴、采访调研、课程整理三选一作为主打。
2. 首版是强结构还是强自然语言：
   我建议优先强结构。
3. 首版是否必须纯端侧：
   我建议主链路必须本地可用，但允许云端辅助兜底。
4. 首版是否带“继续创作”：
   我建议至少带一个，例如“生成行动清单”或“生成内容提纲”。

## 4. 我建议你的默认答案

- 主场景：`头脑风暴 / 会议创意整理`
- 输出形态：`结构化纪要优先`
- 技术路线：`Qwen 本地部署优先，云端只辅助`
- 参赛策略：`先跑通本地模型，再补多模态强化和性能优化`

## 5. 下一步最值得做的产物

如果我们继续推进，最有价值的三个交付物是：

1. Android 首版信息架构图
2. 结构化纪要 JSON schema v1
3. 本地模型接入与设备分层方案

## 6. 验收标准

当下面这些都满足时，说明项目已经从“想法”进入“可做”状态：

- 用户能在手机上独立完成一次多模态输入
- 系统能在本地稳定返回结构化纪要
- 结果页不是纯文本堆砌，而是有明确区块和可编辑内容
- 整个演示流程在几分钟内讲清楚
- 云端关闭后核心流程仍能运行
