# 真实模型接入步骤

这份工程现在已经适配官方 `MNN-LLM` 导出的真实模型包。

## 1. 你最终要拿到什么

以当前单模型方案里的 `Qwen3-VL-2B` 为例，模型目录至少应包含：

```text
<model_dir>/
  tokenizer.txt
  llm.mnn
  llm.mnn.weight
  llm_config.json
  config.json
  visual.mnn
  visual.mnn.weight
```

这套结构来自官方 `llmexport.py` 导出结果。

## 2. 导出模型

先准备一个本地 `Qwen` 模型目录，比如从 `ModelScope` 或 `Hugging Face` 拉到本机：

```bash
scripts/export_qwen_mnn.sh --model-path /absolute/path/to/Qwen-model
```

如果你想自己指定输出目录：

```bash
scripts/export_qwen_mnn.sh \
  --model-path /absolute/path/to/Qwen-model \
  --dst-path /absolute/path/to/exported-qwen-mnn
```

## 3. 把导出目录接入工程

导出完成后，直接执行：

```bash
scripts/link_qwen_model_dir.sh --model-dir /absolute/path/to/exported-qwen-mnn
```

它会自动：

1. 检查当前单模型方案所需文件是否齐全
2. 确认目录可作为 `qwen-vl-2b-instruct-mnn` 的导出结果使用

## 4. 验证工程

```bash
source ~/.zshrc
./gradlew :app:assembleDebug
```

启动 App 后，首页重点看这几项：

- `真实 MNN 已链接`
- `模型安装状态`
- `模型目录探测`
- `会话打开结果`

## 5. 当前工程已经能识别的真实文件名

- tokenizer: `tokenizer.txt` 或 `tokenizer.json`
- model graph: `llm.mnn` 或 `model.mnn`
- runtime config: `config.json`

对于 `MNN-LLM` 官方导出包，当前默认按下面这些文件作为完整包来接：

- `tokenizer.txt`
- `llm.mnn`
- `llm.mnn.weight`
- `llm_config.json`
- `config.json`
- `visual.mnn`
- `visual.mnn.weight`

## 6. 下一步

当这套真实目录接入成功后，当前 JNI 已经会优先走“真正读取 `config.json` 并创建 `Llm` 会话”；如果只拿到了 `libMNN.so` 但缺少 `llm/llm.hpp` 头文件，则会自动退回到 `Interpreter` 级别的图加载校验。
